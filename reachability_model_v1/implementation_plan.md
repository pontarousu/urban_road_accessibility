# 都市道路ネットワークの定量分析と最適化・建設計画

本計画は、OpenStreetMap (OSM) および `osmnx` を用いて、都市の非効率な空間構造を可視化し、「どこに新しい道路や橋を作るのが最も効率的か」を導き出すプロジェクトです。

---

## 1. 【完了】Phase 1: 面積算出手法の検証と確立
コスト・ラスタ歩行シミュレーション空間（Fast Marching Method）を用いることで、「ポリゴンの嘘（川などの非侵入領域を跨いでしまう問題）」を数学的に完璧に排除し、都市から山間部まで普遍的に適用できる到達面（IoU）の計算基準を確立しました。

## 2. 【完了】Phase 2: 都市スケールへの拡張とヒートマップの作成
金沢市全域（10km四方）を対象に、数百地点での計算を並列処理するスクリプトを構築しました。結果として出力されたヒートマップにより、川や鉄道に起因するローカルな交通分断ポイント（赤い目詰まり地点）が明確に可視化されました。

---

## 3. 【Next】Phase 3: 新規道路建設の最適化アルゴリズム (Micro Optimization)

### Phase 3 の目的
Phase 2のヒートマップによって浮き彫りになった「赤い点（交通のボトルネック・分断地域）」をターゲットとし、そこに**仮想的な新しい道路（エッジ）を1本追加することで、どれだけ通信効率（IoU）が跳ね上がるかをシミュレーション**し、最も投資効果の高い（道を作るべき）場所を特定します。

### 実行計画 (Execution Steps)

1. **ターゲット（ボトルネック）の抽出**
   * Phase 2のヒートマップ（金沢市）の実行結果から、周辺と比較して極端にIoUが低い「レッドゾーン」をいくつかピックアップします（例：犀川や浅野川の周辺、あるいは特定の踏切・行き止まりが多い住宅街など）。

2. **方向別歪みの再計算（ターゲット周辺の弱点探索）**
   * ターゲット地点において、「どの方向に進めないせいでIoUが下がっているのか」を探索します。

3. **仮想エッジ（橋・道路）の自動生成シミュレーション**
   * ターゲット周辺において、「物理的な直線距離は近いが、ネットワーク上の移動距離は非常に遠い（迂回を強いられている）」ノードのペア上位を探し出します。
   * そのノード間に「仮想エッジ（擬似的な新しい道路や橋）」をネットワークに挿入します。

4. **投資対効果（ROI）の測定**
   * 仮想エッジ追加後のネットワークで再度コスト・ラスタによるIoUを計算します。
   * **`ROI = 改善したIoUの差分 (Delta IoU) / 追加した道路の長さ`**
   * このスコアに基づき、最も「短く建設できて、最も周辺の効率を改善する夢のプレイス」をランキング化します。

5. **最適な建設地点の可視化**
   * マップ上に「元のボトルネック」「追加すべき新しい道路の線（赤色など）」「追加後にどれだけ到達圏が広がるかの比較画像」を出力します。

> 2. **ROIの定義:** 「改善幅 / 道路の長さ」を投資対効果と定義していますが、「長い橋を作っても劇的に改善するならOK」とするか、「短い橋で効果が高いものを絶対優先」するか、この定義で問題ないかご確認をお願いします。（基本は「改善幅 / 長さ」のコストパフォーマンスをベースとします）
