# ヒートマップ生成と最適化ログ (Phase 2 & 3)

このファイルは、コスト・ラスタ手法を用いた「都市全体のネットワーク効率スコア（IoU）のヒートマップ作成」および「新規道路の建設予測」に関するログ・結果を記録します。

## 対象都市：金沢市（石川県）
金沢の城下町特有の詳細な街区と、それを囲む犀川・浅野川のバリア、および周辺の山間部という特徴を用いて、都市構造の非効率性を可視化します。

## アプローチと設定（パラメータ）
* **分析範囲:** 金沢市中心部から半径 5000m（約 10km × 10km）
* **サンプリング間隔:** 500m ピッチで起点を配置（理論上 $20 \times 20 = 400$ 程度の起点）
* **計算アルゴリズム:** 手段B（コスト・ラスタ歩行シミュレーション：解像度10m）
* **評価指標 (IoU):** 到達面ピクセル面積 / ユークリッド面積 ($ \pi r^2 $)
* **想定到達限界距離:** 1500m（徒歩で約20分弱のスモールスケール圏内で評価）

## ログ・履歴

### [Phase 2 Execution] 金沢市スケールでの並列処理IoUヒートマップ作成 (2026-04-10)

`Joblib`による並列処理（ProcessPoolExecutor）を用いて、金沢市周辺の大規模なヒートマップ生成に成功しました。
*   中心部から半径5kmのエリア上に500m間隔で361箇所（19 x 19）の測定ポイントを展開。
*   巨大なマスター・コストラスタ（10mメッシュ）をメモリマップで共有し、全361地点の「コスト計算ダイクストラ」を8並列で一気に行いました（計算コア部分の処理時間はわずか23秒！）。

![Kanazawa Heatmap](/Users/pontarousu/.gemini/antigravity/brain/6e76f7ef-41b2-415c-92db-2f694ba76e8e/heatmap_kanazawa.png)

#### 視覚的考察（都市の特性の可視化）
*   **高効率エリア（グリーン付近）**: やはり兼六園・香林坊周辺や、少し外れた区画整理された住宅街など、道路構造が比較的均質でグリッドに近い場所でIoUが高くなっています。
*   **低効率エリア・ボトルネック（レッド付近）**:
    1.  **川・鉄道の周辺**: 犀川や浅野川、あるいはJR北陸本線に沿ったエリアで、スコアがガクッと落ちている場所（赤点）が見られます。これは「空間的には近いのに、橋や踏切がないため大きく迂回させられている」ことを示唆しています。
    2.  **山間部・周辺部**: 東部の山に近づくにつれ、道が極端に少なくなるためIoUはレッドに近づきます。

これは都市の「交通のバリア」をあぶり出すという点で、非常に価値のある一枚となりました。

## Phase 3 ログ: 新規道路建設の最適化

### [Phase 3 Execution] 金沢市における「最小コスト・最大効果」最適化シミュレーション (2026-04-10)

**目標関数:**
$$\text{ROI} = \frac{\sum_{i \in \text{影響圏}} \Delta\text{IoU}_i}{L_{\text{edge (m)}}}$$

**アルゴリズムの手順:**
1. ベースラインIoUの下位30パーセンタイル（IoU ≤ 0.076）を「ボトルネック」として定義 → 108地点を抽出
2. ボトルネック付近のノード群（6057ノード）からランダムサンプリング300個を対象に、「Euclidean ≤ 300m かつ ネットワーク迂回比 ≥ 3.0倍」のペアを探索 → **19候補の仮想エッジ**を発見
3. 各候補ごとにコストラスタ面へ仮想線を焼き込み、影響圏1500m内の全サンプル地点でIoUを再計算してΔIoUを算出
4. ROI（= ΔIoU合計 / エッジ長）でランキング化

**結果 Top-10 ランキング:**

**Rank 1の特徴:** わずか **116mの道路（橋）** を架けるだけで、**9.6倍もの迂回を解消**し、最高のコストパフォーマンスを達成。

### [Visualization Fix] optimization_result / best_beforeafter の修正 (2026-04-11)

ユーザーからの指摘を受け、以下の2点を修正・再生成しました。

**① optimization_result：周辺が無色になる問題**
- **原因:** delta（ΔIoU）の実際の値は0.002〜0.01と非常に小さい。これをBluesカラーマップの`vmax=0.3`で描画すると、ほぼ透明（ゼロ同然）に見えてしまっていた。
- **修正:** delta > 0 の点に「After IoUの絶対値」を`Blues`カラーマップで描画するように変更。`vmax`を実際の最大IoU値に合わせてダイナミック化し、改善エリアが明確に青くなるようにした。

**② best_beforeafter：カラーバーと文字の重なり**
- **修正:** `fig.add_axes([0.92, 0.15, 0.015, 0.7])`で独立した領域にカラーバーを配置し、地図と完全に分離。figサイズ(22×10)拡大・`subplots_adjust`で十分な余白を確保。タイトル位置も`y=1.02`に調整。
