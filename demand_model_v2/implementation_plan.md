# 道路需要と動的エージェント・シミュレーション (Demand Model v2)

**最終目標: 「有効な新規道路の敷設アルゴリズム」の構築**

「有効性」= 新規道路を追加することで、ODペア間の到達時間をどれだけ短縮できるか。
$$\Delta T = T_{\text{before}} - T_{\text{after}}$$

---

## 完了済みステップ

### Step 1: グラフ上の単一パケット・ナビゲーション ✅
ラスタ法からグラフ法への転換。Dijkstra法による最短経路探索。

### Step 2: パケット分裂と True Fork Node ✅
複数目的地によるパケット分裂。OSMノード群から「真の分岐点」を特定するアルゴリズム。

### Step 3: BPR関数による渋滞モデリング ✅
エッジごとのCapacity設定、BPR関数による動的コスト計算。

### Step 3.5: 逐次シミュレーション・エンジン ✅ ← 現在地
交差点ごとに貯留容量を判定し、パケットが自発的に分裂する逐次エンジン。

**実装済みの修正:**
- v1: 基本的な逐次ループ（フローベース容量、Volume累積）
- v2: Volume退出処理 + 貯留容量（次元統一）
- v3: 段階的投入（Gradual Injection: 100人/step × 15ステップ）

**現在の結果:** 1500人 →  65ステップで全員到着、1000パケットに自然分裂

---

## 今後のロードマップ

### Step 4: シミュレーション結果の検証と可視化改善
**目的:** 現在の逐次エンジンが物理的に妥当な結果を出しているかを定量的に検証する。

- [ ] 各パケットの到達時間（ステップ数）のヒストグラムを描画
- [ ] エッジ交通量の時系列グラフ（どのエッジがいつ混むか）
- [ ] 渋滞発生箇所のヒートマップ（どの交差点で分裂が多いか）
- [ ] injection_rateの感度分析（50, 100, 200, 500で比較）
- [ ] アニメーション品質の最終調整

### Step 5: 複数ODペアの同時シミュレーション
**目的:** 単一のスタート→ゴールではなく、都市全体に需要を分散させる。

- [ ] ネットワーク上のランダムな複数ODペアを生成
- [ ] 各ODペアから独立にパケットを段階投入
- [ ] 異なるODペアのパケットが同じエッジで干渉する「相互渋滞」の表現
- [ ] 総到達時間 $T$ の統計量（平均、分散、最大）の算出

### Step 6: 道路新設の評価関数 $\Delta T$ の実装
**目的:** 「道路を1本追加したらどれだけ改善するか」を定量化する。

- [ ] グラフに仮想エッジを追加する `add_virtual_road(G, node_a, node_b, highway_type)` 関数
- [ ] Before/Afterシミュレーションの自動実行パイプライン
- [ ] $\Delta T$ の算出と可視化（どの道路が最も効果的か）
- [ ] 道路追加のコスト概念の導入（長さ × 車線数 ∝ 建設費用）

### Step 7: 道路敷設の最適化アルゴリズム
**目的:** $\Delta T$ を最大化する道路配置を探索する。

- [ ] 候補エッジの生成方法の設計（ボトルネック交差点の近傍など）
- [ ] グリーディ探索 or 遺伝的アルゴリズムで最適配置を探索
- [ ] 予算制約下での最適化（コスト $\leq$ 予算 のもとで $\Delta T$ 最大化）
- [ ] 既存の都市計画手法との比較評価

### Step 8: リザバーコンピューティング（RC）への統合（発展課題）
**目的:** 交通動態の時系列データをRCに入力し、都市の効率性パターンを学習する。

- [ ] エッジ交通量の時系列サンプリング（各ステップの全エッジvolumeを記録）
- [ ] Echo State Networkの構築と入力信号の設計
- [ ] 渋滞予測や最適道路配置の学習可能性の検証
